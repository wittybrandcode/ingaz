## المرحلة 2 — Backend Authorization

### التغييرات

#### 2.1 إعادة كتابة `server/src/middleware/auth.ts`
- **`authenticate`**: استخراج `is_manager` من JWT ووضعه في `req.user`
- **`authorize`**: إضافة bypass للمدير (`isManager`) — يمر دون فحص
- **`authorizePermission`**: دوال جديدة للتحقق من صلاحية محددة عبر `role_permissions`
- **`hasPermission`**: فحص دور المستخدم في `role_permissions`
- **`requireManager`**: دالة جديدة — للمسارات التي يحتاجها المدير فقط

#### 2.2 تحديث `server/src/services/AuthService.ts`
- إضافة `isManager` إلى استعلامات `login()` و `me()`

#### 2.3 تحديث `server/src/services/UserService.ts`
- إضافة `permissions` (مصفوفة صلاحيات المستخدم) إلى `list()`

#### 2.4 تحديث كل ملفات المسارات
- **`users.ts`**: استبدال `authorize(ROLES.ADMIN)` ← `requireManager`
- **`roles.ts`**: استبدال `authorize(ROLES.ADMIN)` ← `requireManager`
- **`projects.ts`**: استبدال `authorize(ROLES.ADMIN, ROLES.DEPUTY)` ← `authorizePermission('projects.*')`
- **`tasks.ts`** و **`subtasks.ts`** و **`comments.ts`** و **`upload.ts`** و **`warnings.ts`**: استبدال `authorize(ROLES.ADMIN, ROLES.DEPUTY)` بـ `authorizePermission` أو `requireManager`

#### 2.5 تحديث `server/src/constants.ts`
- حذف `ROLES.DEPUTY` و `ROLES.EMPLOYEE`
- حذف `ROLES_VALUES.DEPUTY` و `ROLES_VALUES.EMPLOYEE`

#### 2.6 تحديث `ServiceContext`
- إضافة `isManager` إلى واجهة `ServiceContext` في `BaseService.ts`

#### 2.7 تحديث الخدمات الرئيسية
- `SubtaskService.ts`: إضافة فحص `isManager` في `canChangeStatus` و `create`
- `TaskService.ts`: إضافة فحص `isManager` للسماح بإنشاء المهام
- `ProjectService.ts`: إضافة فحص `isManager` في عمليات التعديل والحذف
- `CommentService.ts`: إضافة فحص `isManager`
- `UploadService.ts`: إضافة فحص `isManager`
- `DeadlineService.ts`: إضافة فحص `isManager`
