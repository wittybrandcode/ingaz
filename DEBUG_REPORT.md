# Technical Debug Report — Bigg (Ingaz)

**Status:** Analysis Complete — Remediation Recommended

## 1. Executive Summary
The "Bigg" application is a mature, full-stack Arabic task-management system built with React (Vite) and Node.js (Express + Drizzle ORM). While the codebase is modern and well-structured, a recent technical audit and deep-dive analysis have identified several critical issues that impact security, performance, and scalability.

**Key Findings:**
- **Security:** Several high-risk vulnerabilities, including a hardcoded JWT fallback secret and potential XSS vectors.
- **Performance:** Significant bottlenecks in core API endpoints due to lack of pagination and inefficient query patterns (N+1 queries).
- **Code Quality:** The project lacks comprehensive test coverage and contains several "god components" that violate the Single Responsibility Principle (SRP).

---

## 2. Identified Issues & Severity

### 🔴 Critical Severity
| ID | Issue | Location | Impact |
|---|---|---|---|
| **S1** | **Hardcoded JWT Secret Fallback** | `auth.ts`, `index.ts` | Allows an attacker to forge JWT tokens if the environment variable is not set. |
| **P1** | **N+1 Query in Subtask Assignees** | `SubtaskService.ts` | API performs a separate database query for each subtask to fetch its assignees. |
| **T1** | **Near-Zero Test Coverage** | Entire Project | Changes cannot be verified safely, leading to a high risk of regressions. |

### 🟡 High Severity
| ID | Issue | Location | Impact |
|---|---|---|---|
| **S2** | **Stored XSS in Subtask Descriptions** | `subtasks.ts` | User-provided descriptions are not sufficiently sanitized on the server. |
| **P2** | **Hardcoded Pagination in Project Details** | `ProjectService.ts` | `GET /projects/:id` is hardcoded to return only the first page of tasks. |
| **A1** | **Critical Accessibility Gaps** | `ProjectDetail.tsx` | Missing ARIA labels and keyboard navigation on interactive elements. |

---

## 3. Recommendations & Action Plan

### Immediate Actions
1. **Fix S1:** Remove the hardcoded fallback for `JWT_SECRET`. Ensure the server crashes if it's missing.
2. **Fix S2:** Implement strict server-side sanitization for all user-generated content.
3. **Fix P1:** Batch fetch subtask assignees using the existing bulk helper.

### Short-Term
1. **Fix P2:** Update `ProjectService.ts` to respect `page` and `pageSize` parameters from the request.
2. **Initiate Testing:** Set up a Vitest suite for core business logic.
